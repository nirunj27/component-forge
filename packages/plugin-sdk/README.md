# Plugin SDK

Extend Component Forge with custom generators, transforms, and validation — without forking the core engine.

## Concepts

| Piece | Role |
|-------|------|
| **Plugin** | A package that registers hooks |
| **Hooks** | `beforeGenerate`, `transformFile`, `afterGenerate` |
| **forge.config.ts** | Lists plugins to load at generate time |

## Pipeline

```text
Schema
  → beforeGenerate (all plugins)
  → core React templates
  → transformFile (per file, per plugin)
  → afterGenerate (all plugins)
  → written files
```

## Create a plugin

```typescript
// plugins/my-plugin/src/index.ts
import { definePlugin } from "@component-forge/plugin-sdk";

export function myPlugin() {
  return definePlugin({
    name: "@acme/forge-my-plugin",
    version: "1.0.0",
    description: "What this plugin does",
    hooks: {
      transformFile(file, { schema }) {
        if (!file.filename.endsWith(".tsx")) return file;
        return {
          ...file,
          content: file.content.replace(
            "export function",
            "// generated by acme\nexport function",
          ),
        };
      },
    },
  });
}
```

## Register a plugin

```typescript
// forge.config.ts
import { defineConfig } from "@component-forge/plugin-sdk";
import { testIdPlugin } from "@component-forge/plugin-testid";

export default defineConfig({
  plugins: [testIdPlugin()],
});
```

## CLI

```bash
pnpm plugins          # list loaded plugins
pnpm generate:all     # runs with plugins from forge.config.ts
```

## Built-in example: `@component-forge/plugin-testid`

Injects `data-testid="button"` (kebab-case component name) into generated `.tsx` files for easier E2E testing.

## Hook reference

### `beforeGenerate(schema)`

Mutate the schema before templates run. Return a new schema object.

### `transformFile(file, context)`

Transform a single output file. Called for every file × every plugin.

### `afterGenerate(context)`

Replace or extend the full file list. Return a new `GeneratedFile[]`.

## Interview talking point

> Plugins let teams enforce org-specific rules (test IDs, file headers, extra docs) as code — the core generator stays stable while conventions evolve per team.
